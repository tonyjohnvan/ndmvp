App Setup

Credentials for testing your app

Apps often require a valid login to access all features, we recommend you provide one so that NimbleDroid can access everything real users will see. You will probably want to set up the user account with data that will facilitate testing, like sample friends, favorites, etc. We will store and use these credentials, so don't provide an account that has access to proprietary or sensitive information.

Upload your app

Let's kick things off by uploading your app. We'll add it to our profile queue while you continue NimbleDroid setup. Your APK file is kept private and once profiled we'll delete it.

Automatically upload builds (optional)

To get the full power of NimbleDroid, we recommend automatically uploading apks as part of your build process.

- Generate performance history for your development process

- Save time manually uploading builds

- Catch issues earlier in the release proces

Setup Gradle plugin

You will always have the ability to upload either way in the future.

1. Add NimbleDroid plugin snippet to build.gradle

The NimbleDroid plugin will automatically be downloaded as a dependency from plugins.gradle.org. This syntax works for Gradle 2.1 and up. For older versions, see https://plugins.gradle.org/plugin/com.nimbledroid.profiler 

plugins {
  id "com.nimbledroid.profiler" version "1.0.8"
}

2. Add API key snippet to build.gradle

Most company security policies forbid adding credentials directly to build.gradle. You should set an API key variable elsewhere, this could be in the gradle.properties file, an environment variable, or wherever you securely store credentials.

Replace the instruction string with a reference to that variable.

Your API key is 156a495a52ab4d83344a197d22e270c2

nimbledroid {
  apiKey "replace with statement to access key variable"
}

3. Add ndUpload to your Jenkins Gradle tasks

If you are using Jenkins, you will need to add the NimbleDroid task ndUpload to your Jenkins Gradle tasks. The task must be added after the step that produces your APK. For example:

clean assembleRelease ndUpload ndGetProfile

This will upload your APK with every build. Don't forget to save your Jenkins project after making changes.

4. Test your setup

To verify your setup is working, go to the command line where you would normally run Gradle commands and run: 

# gradle clean assembleRelease ndUpload

If you encounter any issues during setup, please send us a message and we'll try help.
